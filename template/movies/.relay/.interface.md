# Movie Repository Browser

A structured file-based movie repository with automated validation and indexing.

## Overview

This repository provides a standardized way to store and organize movie metadata using a file-based structure. It includes automatic validation through Git hooks to ensure data integrity and consistency.

## File Structure

### Primary Data Files
Movie metadata is stored in JSON files following this pattern:
```
data/<year>/<title_trimmed>/meta.json
```

Each file contains:
- `title`: Official movie title (string)
- `imdb`: IMDB ID (format: tt followed by digits)
- `ipfs`: IPFS content identifier (string)

### Index Files
Movies are indexed by director and genre for easy browsing:
```
byDirector/<director>/<movie>.json
byGenre/<genre>/<movie>.json
```

Index files contain:
- `movie`: Relative path to movie data (format: `<year>/<title_trimmed>`)

## Title Normalization

Movie titles are normalized to filesystem-safe names using these rules:
1. Trim leading/trailing whitespace
2. Convert to lowercase
3. Replace spaces and non-alphanumeric sequences with single hyphens
4. Remove unsafe filename characters

Example: `"The Lord of the Rings: The Fellowship of the Ring"` → `"the-lord-of-the-rings-the-fellowship-of-the-ring"`

## Validation

### Schema Definition
The repository structure is defined in `schema.yaml`, which specifies:
- File path patterns with variable placeholders
- JSON schema validation for each file type
- Metadata rules and examples

### Git Hooks
Automated validation is enforced through Git hooks:

#### Pre-commit Hook (`precommit.js`)
- Validates staged files before commits
- Requires Node.js and js-yaml
- Fails commit if validation errors found

#### Pre-receive Hook (`prereceive.js`)
- Server-side validation for pushed commits
- Ensures repository integrity at the server level
- Rejects invalid pushes

## Usage

### Adding a Movie

1. Create the movie directory:
   ```bash
   mkdir -p data/2023/the-dark-knight
   ```

2. Create the metadata file:
   ```json
   {
     "title": "The Dark Knight",
     "imdb": "tt0468569",
     "ipfs": "Qm..."
   }
   ```

3. Add index entries:
   ```bash
   mkdir -p byDirector/christopher-nolan
   echo '{"movie": "2008/the-dark-knight"}' > byDirector/christopher-nolan/the-dark-knight.json

   mkdir -p byGenre/action
   echo '{"movie": "2008/the-dark-knight"}' > byGenre/action/the-dark-knight.json
   ```

4. Commit the changes (validation will run automatically)

### Browsing Movies

- **By Year**: List directories in `data/`
- **By Director**: Browse `byDirector/` directories
- **By Genre**: Browse `byGenre/` directories

Each index file points to the actual movie data.

## Setup

1. Copy hooks to Git:
   ```bash
   cp .relay/precommit.js .git/hooks/pre-commit
   chmod +x .git/hooks/pre-commit

   cp .relay/prereceive.js .git/hooks/pre-receive
   chmod +x .git/hooks/pre-receive
   ```

2. Install dependencies:
   ```bash
   npm install js-yaml
   ```

## Customization

The system is designed to be generic:
- Modify `schema.yaml` to change file patterns or validation rules
- Update hook scripts for different validation logic
- Adapt the structure for other types of repositories

## Validation Rules

### Primary Files
- `title`: Non-empty string
- `imdb`: Must match `^tt\d+$`
- `ipfs`: Non-empty string

### Index Files
- `movie`: Must match `^\d{4}/[a-z0-9\-]+$`

Invalid files will be rejected by the Git hooks with detailed error messages.
